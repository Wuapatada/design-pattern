# Design Patterns Katas – Student Edition

Repositorio de práctica para implementar y refactorizar sistemas usando patrones de diseño en Java.

El objetivo de este repositorio es **practicar el razonamiento de diseño**, no solo copiar código.  
Cada kata representa un reto donde primero se analiza un código con problemas y después se refactoriza aplicando un patrón específico.

---



## ¿Cómo trabajar cada kata?

1. Leer el `README.md` principal del patrón para entender:
   - El contexto del sistema.
   - La explicación del patrón.
   - Cuándo aplicarlo.
   - Los beneficios y principios SOLID involucrados.

2. Ir a la carpeta `challenge/` del patrón:
   - Leer las **Instrucciones del reto**.
   - Analizar el **Código a Refactorizar**.
   - Identificar los problemas de diseño mencionados.
   - Intentar proponer tu propia solución.

3. Revisar la carpeta `solution/` solo después de intentar el reto:
   - Leer el `README.md` de la solución.
   - Comparar tu diseño con el propuesto.
   - Ejecutar el código y revisar la salida esperada.
